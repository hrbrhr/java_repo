Давайте в свободной форме опишем как будет реализована
и работать наша реализация списка.

Предлагаем реализовать список на основе массивов.

Например: в начале создаем пустой массив. В массиве будем
хранить элементы списка.

При добавлении нового элемента в список создаем массив
большего размера, копируем в него элементы из старого массива
и добавляем новый элемент последним в список.

При удалении элемента из списка делаем все наоборот.
Создаем массив меньшего размера, копируем в него все элементы
кроме того что удаляем. "Забываем" ссылку на старый массив
и делаем его доступным для GC (Garbage Collector).

На подобном механизме работает популярная реализация
списка под названием ArrayList (конечно без учета разных оптимизаций).
